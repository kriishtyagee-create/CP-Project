Architecture & Deployment

This system operates on a separated frontend and backend API model.

Backend (Flask) Deployment

  • Deployed via Render or Heroku.
  • Dependencies: `Flask`, `Flask-SQLAlchemy`, `PyJWT`, `Flask-Cors` (via requirements.txt).
  • Database: Built-in SQLite for portability.
  • Run via `gunicorn app:app`.

Frontend Deployment

  • Can be served statically via Netlify or Vercel.
  • Currently served securely via Flask `render_template`.
  • Communicates via standard DOM Fetch commands.
Sample Endpoints:
GET /api/attendance
POST /api/attendance
POST /api/login
